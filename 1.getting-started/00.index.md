Welcome to the dxflow Hub! This section will help you understand how dxflow works and how to make the most of the workflow catalog.

## What is dxflow Hub?

The dxflow Hub is a curated collection of production-ready workflows for scientific computing, data science, and engineering. Workflows are provided primarily as Docker Compose configurations and can be deployed across multiple container runtimes—Docker, Podman, Singularity, and Apptainer—through dxflow's unified interface.

## Why Use dxflow Hub?

**Runtime-Agnostic**: Workflows run on Docker, Podman, Singularity, and Apptainer through a unified interface.

**Ready-to-Deploy**: All workflows are tested and production-ready with complete configurations.

**Unified Management**: Deploy, monitor, and manage all workflows through a single interface - CLI, API, or web UI.

**Flexible Infrastructure**: Run on any infrastructure - cloud VMs, HPC clusters, edge devices, or your laptop.

**Scientific Focus**: Curated workflows for genomics, molecular dynamics, structural biology, data science, and CFD.

## Deploy

Deploy any workflow with the dxflow CLI:

```bash
# Create and start a workflow
dxflow workflow create --identity <workflow-name> <compose-file>
dxflow workflow start <workflow-name>

# Follow the logs live
dxflow workflow logs --live <workflow-name>
```
